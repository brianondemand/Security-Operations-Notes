### 3.1 SIEM Architecture

**Security Information and Event Management (SIEM)** systems like [[Splunk]], ELK Stack, and [[QRadar]] aggregate and correlate log data from across the organization. They provide a unified view for security analysis, alerting, and reporting.

### 3.2 Data Ingestion and Normalization

This is the process of collecting logs from diverse sources (firewalls, servers, endpoints, cloud services) and converting them into a standard format (normalization) so they can be analyzed together.

### 3.3 Event Correlation

The "secret sauce" of a SIEM. Correlation links seemingly unrelated events from different sources to identify complex, multi-stage attacks. For example, a failed login followed by a successful login from a new country might indicate a credential compromise.

### 3.4 Search Processing Language (SPL)

The query language used to search and analyze data within a SIEM. In [[Splunk]], SPL allows analysts to pipe commands together to filter, transform, and visualize data.

> [!example]- Basic SPL for Brute Force Detection
> ```spl
> index=main action=login status=failed
> | stats count by src_ip
> | where count > 100
> | sort - count
> ```
