## Module 5: Proactive Defense - Threat Hunting & Intelligence


> [!goal] Learning Objectives
> *   Differentiate between reactive alerting and proactive threat hunting.
> *   Utilize threat intelligence to guide hunting hypotheses.
> *   Analyze endpoint data to find adversaries.

---

### 5.1 Threat Hunting

Threat hunting is a **proactive** approach where analysts hypothesize about potential breaches and actively search for adversaries that may have evaded existing security controls. It is based on the assumption that a breach has already occurred.

### 5.2 Threat Intelligence

Threat intelligence is evidence-based knowledge about existing or emerging threats, used to inform defensive decisions. It can be strategic (high-level trends) or tactical (specific [[Indicators of Compromise]]).

### 5.3 Indicators of Compromise (IOCs)

IOCs are forensic artifacts observed on a network or in an operating system that indicate a potential intrusion. Common IOCs include:
- Suspicious IP addresses and domain names.
- File hashes (MD5, SHA256) of known malware.
- Registry keys or file paths associated with malware.
- Network signatures (e.g., specific HTTP user agents).

### 5.4 Open Source Intelligence (OSINT)

Using publicly available tools to enrich IOCs and gather context. Key tools include:
- **[[VirusTotal]]:** Scans files and URLs against multiple antivirus engines.
- **[[AbuseIPDB]]:** Checks if an IP address has been reported for malicious activity.
- **[[Shodan]]:** A search engine for internet-connected devices.
